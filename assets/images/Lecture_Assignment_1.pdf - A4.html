Action Selection Methods


Multi-Arm Bandit (MAB) problem


The Multi-Arm Bandit (MAB) problem is a classic decision-making problem in
which an agent must choose between multiple actions (or arms) to maximize
its total reward over time. Each arm provides a different, unknown reward
distribution, and the challenge is to find the best one without wasting too
many trials on bad options.


Why is it challenging?


The key difficulty in MAB is the Exploration vs. Exploitation :

  • Exploration: Testing different arms to discover which one has the best
    long-term reward.
  • Exploitation: Choosing the best-known arm to maximize immediate
    reward.

If an agent explores too much, it wastes time on suboptimal arms. If it exploits
too early, it might miss better options. Balancing these two is the hardest part
of the problem.


How is it related to Reinforcement Learning (RL)?


The MAB problem is a simplified version of Reinforcement Learning (RL). In
RL, an agent must make sequential decisions in a more complex
environment while learning from rewards. Many RL algorithms, such as Q-
learning and Policy Gradient methods, build on the concepts of MAB.

For example, in robotic control, an RL agent must learn which actions lead to
higher rewards (better movements) while still trying new actions to improve.

ε Value

Effect

High

The agent explores often, finding hidden rewards but taking longer
to converge.

Low

The agent mostly exploits the best action but may miss better
choices.

ε = 0

The agent never explores, so it can get stuck with a suboptimal
action.

Effect of the Epsilon (ε) Value in ε-Greedy Selection


The ε-greedy algorithm is one of the simplest ways to balance exploration
and exploitation. It works by:

  • With probability ( 1 - ε ): select the best-known action.
  • With probability ( ε ): choosing a random action to explore.


Effect of Different ε Values

Best Practice:


A good approach is to start with a high ε and gradually reduce it over time,
allowing more exploration early on but focusing on exploitation later.

Comparing Action Selection Methods

Why Use UCB Instead of ε-Greedy?


ε-Greedy is basic but kinda wasteful since it picks random actions
sometimes, even when they’re not good. Most of the time, it chooses the
best-known action, but with a small chance (ε), it just explores randomly. That
can slow things down because it might keep trying bad options.

UCB is way smarter. It doesn’t just explore randomly it prioritizes actions that
haven’t been tried much but seem promising. Instead of guessing, it
calculates confidence levels and picks actions that could have high rewards
but need more testing. This way, it balances trying new things and sticking
with what works.
